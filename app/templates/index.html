Parameters

Learning rate · entropy · epsilon — tik op (i) voor uitleg per veld

Learning rate (i)Hoe groot elke gewichtsupdate is. Een lage waarde (bijv. 1e-5) betekent voorzichtig, langzamer leren en kleinere sprongen per train-chunk — minder risico op instabiliteit.
—
Entropy (i)Policy-entropy: hoe 'gespreid' de actiekansen zijn. Lager = kortere/kantere strategie. Een streepje (—) betekent: in dit tick is geen verse entropy-waarde binnengekomen; geen probleem als de rest wél vult.
—
Epsilon (explore) (i)Exploratie: welk deel van de tijd er extra toeval op de acties wordt gemengd (PPO/SB3) om nieuwe situaties te zien. Hangt samen met je RL_EXPLORATION_*-instellingen in de worker.
—
Discount factor (i)γ (gamma): hoe zwaar toekomstige beloningen meetellen t.o.v. directe. Richting 1,0 = het model let meer op lange termijn.
—
Batch size (i)Aantal ervaringen per gradient-update (typisch PPO/SB3). Samen met learning rate bepaalt dit hoe ruig of glad elke update is.
—
Global steps (i)Totaal aantal omgevings-stappen (global_step_count): hoeveel train-stappen PPO al heeft gemaakt. Stijgt als er echte updates draaien. In NL-notatie is een punt vaak duizendtallen (bijv. 217.088 = tweehonderdzeventienduizend).
—
RL Confidence (i)Geschatte zekerheid van het beleid (meestal 0–1). Gaat over het netwerk, niet over hoeveel er vandaag getrade is.
—
Model Version (i)Interne versie-label van het geladen model (build/checkpoint-tag). Geen marktprijs en geen garantie over winst.
—

Prediction Error (TD-Error)

Value loss per PPO-update stap — daalt naarmate het model beter leert (x-as = globale trainingsstappen)

Markt (i)Het handelspaar waarvoor dit model actief is, bijv. BTC-EUR. Model (i)Of het neurale netwerk momenteel geladen en actief is. ✓ = geladen; — = nog niet beschikbaar. Oud (min) (i)Model Freshness: aantal minuten geleden dat dit model voor het laatst getraind of bijgewerkt is. Lager = verser. Replay (i)Aantal opgeslagen ervaringen in de replay-buffer waarmee de AI leert. Meer = meer leervariatie. BUY% (i)Kans op een BUY-beslissing per de huidige policy-softmax. BUY + HOLD + SELL = 100%. HOLD% (i)Kans op HOLD (geen actie). Hoog HOLD = de bot ziet geen overtuigend signaal voor kopen of verkopen. SELL% (i)Kans op een SELL-beslissing. BUY + HOLD + SELL = 100%.
